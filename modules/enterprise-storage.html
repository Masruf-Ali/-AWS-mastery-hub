Module 19: Enterprise Storage

19.1 Amazon FSx

Amazon FSx provides fully managed third-party file systems with the native compatibility and feature sets for your workloads. It eliminates the administrative overhead of managing file servers and storage, allowing you to use familiar file systems in the cloud.

FSx for Windows File Server:

  • Fully Managed: Provides a fully managed native Microsoft Windows file system.
  • SMB Protocol: Accessible via the Server Message Block (SMB) protocol.
  • Active Directory Integration: Seamlessly integrates with Microsoft Active Directory for user authentication.
  • Use Cases: Lift-and-shift Windows applications, home directories, shared file storage.

FSx for Lustre:

  • High-Performance File System: Optimized for compute-intensive workloads like HPC (High-Performance Computing), machine learning, and media processing.
  • Scalable: Scales to hundreds of gigabytes per second of throughput and millions of IOPS.
  • Integration with S3: Can be linked to S3 buckets to process data stored in S3.
  • Use Cases: HPC, financial modeling, electronic design automation.

FSx for NetApp ONTAP:

  • Fully Managed ONTAP: Provides a fully managed shared file storage built on NetApp ONTAP.
  • Multi-protocol Support: Supports NFS, SMB, and iSCSI.
  • Advanced Data Management: Offers ONTAP features like snapshots, replication, and data deduplication.
  • Use Cases: Enterprise applications, databases, DevOps, media workflows.

FSx for OpenZFS:

  • High-Performance ZFS: Provides a fully managed native OpenZFS file system.
  • Scalable: Scales to hundreds of thousands of IOPS and gigabytes per second of throughput.
  • Data Management: Offers ZFS features like snapshots, clones, and data compression.
  • Use Cases: Machine learning, electronic design automation, media processing.

19.2 AWS Storage Gateway

AWS Storage Gateway is a hybrid cloud storage service that enables your on-premises applications to seamlessly use AWS cloud storage. It provides a bridge between your on-premises infrastructure and AWS cloud storage, allowing you to store data in AWS for scalability, durability, and cost-effectiveness.

Gateway Types:

  • File Gateway: Stores files as objects in S3, accessible via NFS and SMB protocols.
  • Volume Gateway: Provides block storage to your on-premises applications using iSCSI. It comes in two modes:
    • Cached Volumes: Stores primary data in S3 and retains frequently accessed data locally in a cache.
    • Stored Volumes: Stores primary data locally and asynchronously backs up snapshots to S3.
  • Tape Gateway: Provides a virtual tape library (VTL) interface, allowing you to use your existing backup applications with a cloud-based virtual tape library.

Use Cases: Hybrid cloud storage, backup and recovery, disaster recovery, archiving.

19.3 AWS DataSync

AWS DataSync is a data transfer service that makes it easy for you to automate moving data between on-premises storage and Amazon S3, Amazon EFS, or Amazon FSx for Windows File Server. It simplifies, automates, and accelerates copying large amounts of data.

Key Features:

  • Automated Data Transfer: Automates copying data between various storage locations.
  • Accelerated Transfer: Uses a purpose-built network protocol and parallelization to speed up transfers.
  • Secure: Data is encrypted in transit and at rest.
  • Managed Service: No need to manage transfer infrastructure.
  • Use Cases: Data migration, recurring data transfers, disaster recovery.

DataSync can transfer data at speeds up to 10x faster than open-source tools.
