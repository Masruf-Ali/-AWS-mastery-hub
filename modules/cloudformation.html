Module 21: AWS CloudFormation

21.1 What is AWS CloudFormation?

AWS CloudFormation is a service that helps you model and set up your Amazon Web Services resources so that you can spend less time managing those resources and more time focusing on your applications that run in AWS. You create a template that describes all the AWS resources that you want (like Amazon EC2 instances or Amazon RDS DB instances), and CloudFormation takes care of provisioning and configuring those resources for you.

Key Features:

  • Infrastructure as Code: Treat your infrastructure as code, allowing you to version, review, and automate your infrastructure deployments.
  • Automated Deployments: CloudFormation automates the process of provisioning and updating your infrastructure in a safe and predictable way.
  • Declarative Language: Use a declarative language (JSON or YAML) to describe your desired infrastructure.
  • Change Management: Use change sets to preview how proposed changes to a stack might impact your running resources before you implement them.

21.2 Core CloudFormation Concepts

To use CloudFormation, you need to understand its core components.

  • Template: A JSON or YAML file that defines your AWS infrastructure. The template is the blueprint for your stack.
  • Stack: A collection of AWS resources that you can manage as a single unit. All the resources in a stack are defined by the stack's CloudFormation template.
  • Change Set: A summary of proposed changes to your stack that allows you to see how those changes will affect your running resources before you implement them.
  • StackSet: A way to create, update, or delete stacks across multiple AWS accounts and regions with a single operation.

21.3 How to Set Up a CloudFormation Stack

Creating a CloudFormation stack involves creating a template and then using it to launch a stack.

Basic Setup Steps:

  1. Create a Template: Create a JSON or YAML file that defines the AWS resources you want to create.
  2. Navigate to CloudFormation: In the AWS Management Console, go to the CloudFormation service.
  3. Create Stack: Click the "Create stack" button.
  4. Specify Template: Upload your template file or specify the S3 URL where it is located.
  5. Specify Stack Details: Provide a name for your stack and any required parameters.
  6. Review and Create: Review your stack configuration and click "Create stack".

21.4 Uses of AWS CloudFormation

CloudFormation is a powerful tool for automating your infrastructure deployments.

Common Use Cases:

  • Automating infrastructure deployment: Deploy a complete, multi-tier web application with a single CloudFormation template.
  • Creating reproducible environments: Create identical development, testing, and production environments.
  • Managing infrastructure changes: Use change sets to safely update your infrastructure.
  • Implementing disaster recovery: Quickly recreate your infrastructure in a different region in the event of a disaster.
