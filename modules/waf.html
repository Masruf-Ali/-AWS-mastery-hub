Module 28: AWS WAF

28.1 What is AWS WAF?

AWS WAF is a web application firewall that helps protect your web applications or APIs against common web exploits that may affect availability, compromise security, or consume excessive resources. AWS WAF gives you control over how traffic reaches your applications by enabling you to create security rules that block common attack patterns, such as SQL injection or cross-site scripting, and rules that filter out specific traffic patterns you define.

Key Features:

  • Protection against common attacks: Protects against OWASP Top 10 security risks.
  • Customizable rules: Create your own rules to filter traffic based on IP addresses, HTTP headers, HTTP body, or URI strings.
  • Managed rule groups: Use managed rule groups from AWS or AWS Marketplace sellers to get protection from a wide range of common threats.
  • Real-time visibility: Get real-time visibility into your web traffic and see which requests are being blocked, allowed, or counted.

28.2 Core WAF Concepts

To use AWS WAF, you need to understand its core components.

  • Web ACL (Access Control List): A set of rules that define the traffic patterns to allow or block.
  • Rule: A condition that AWS WAF searches for in web requests. You can create your own rules or use managed rules.
  • Rule Group: A reusable set of rules that you can add to a web ACL.
  • Managed Rule Groups: A pre-configured set of rules managed by AWS or AWS Marketplace sellers to protect against common threats.

28.3 How to Set Up AWS WAF

Setting up AWS WAF involves creating a web ACL and associating it with an AWS resource.

Basic Setup Steps:

  1. Navigate to WAF & Shield: In the AWS Management Console, go to the WAF & Shield service.
  2. Create web ACL: Click the "Create web ACL" button.
  3. Describe web ACL and associate to AWS resources: Provide a name for your web ACL, and choose the AWS resource to associate it with (e.g., an Application Load Balancer, API Gateway, or CloudFront distribution).
  4. Add rules and rule groups: Add rules to your web ACL. You can create your own rules or use managed rule groups.
  5. Set rule priority: Set the priority for your rules to determine the order in which they are evaluated.
  6. Review and create web ACL: Review your web ACL configuration and click "Create web ACL".

28.4 Uses of AWS WAF

AWS WAF is a critical component for securing your web applications and APIs on AWS.

Common Use Cases:

  • Protecting against web attacks: Block common web attacks such as SQL injection and cross-site scripting.
  • Filtering malicious traffic: Filter out traffic from known malicious IP addresses.
  • Geographic blocking: Block traffic from specific geographic locations.
  • Rate-based rules: Protect your applications from brute-force attacks and other volumetric attacks by limiting the number of requests from a single IP address.
