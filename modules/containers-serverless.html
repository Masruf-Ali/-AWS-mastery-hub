Module 15: Containers & Serverless

15.1 Containers Overview

Containers provide a way to package an application with all its dependencies (code, runtime, system tools, libraries, settings) into a single, isolated unit. This ensures that the application runs consistently across different environments, from development to production.

Key Benefits:

  • Portability: Run consistently anywhere.
  • Efficiency: Share the host OS kernel, making them lightweight and fast to start.
  • Isolation: Applications are isolated from each other and the host system.
  • Scalability: Easy to scale up and down.

Docker is the most popular containerization technology.

15.2 Amazon Elastic Container Service (ECS)

Amazon ECS is a fully managed container orchestration service that makes it easy to deploy, manage, and scale containerized applications. ECS integrates deeply with the rest of the AWS platform to provide a secure and easy-to-use solution for running container workloads.

Key Concepts:

  • Cluster: A logical grouping of EC2 instances (or Fargate launch type) where tasks are run.
  • Task Definition: A blueprint for your application, specifying the Docker image, CPU, memory, networking, and other parameters.
  • Task: An instance of a task definition running on a container instance.
  • Service: Maintains a desired number of tasks and can integrate with an Elastic Load Balancer.

15.3 Amazon Elastic Kubernetes Service (EKS)

Amazon EKS is a fully managed Kubernetes service. Kubernetes is an open-source system for automating deployment, scaling, and management of containerized applications. EKS makes it easy to run Kubernetes on AWS without needing to install, operate, and maintain your own Kubernetes control plane.

Key Features:

  • Managed Control Plane: AWS manages the Kubernetes control plane (master nodes), ensuring high availability and patching.
  • Worker Nodes: You provision and manage the worker nodes (EC2 instances) or use Fargate.
  • Open Source: Compatible with standard Kubernetes tools and plugins.
  • Integration: Integrates with other AWS services like IAM, VPC, and Load Balancers.

EKS is ideal for organizations that want to leverage the power and flexibility of Kubernetes while offloading the operational burden of managing the control plane to AWS.

15.4 AWS Fargate

AWS Fargate is a serverless compute engine for containers that works with both Amazon ECS and Amazon EKS. Fargate removes the need to provision and manage servers, letting you focus on building your applications.

Key Benefits:

  • Serverless: No EC2 instances to manage. You only pay for the resources your containers consume.
  • Simplified Operations: AWS handles server provisioning, patching, scaling, and infrastructure management.
  • Cost-Effective: Pay only for the vCPU and memory resources your application requests.
  • Security: Each task or pod runs in its own isolated compute environment.

Fargate is an excellent choice for applications that need to run containers without the operational overhead of managing EC2 instances.

15.5 AWS Lambda

AWS Lambda is a serverless, event-driven compute service that lets you run code without provisioning or managing servers. You pay only for the compute time you consume.

Key Features:

  • Serverless: No servers to manage.
  • Event-Driven: Code runs in response to events (e.g., S3 object upload, DynamoDB stream, API Gateway request).
  • Scalability: Automatically scales to meet demand.
  • Cost-Effective: Pay only for the actual compute time consumed (per millisecond) and the number of requests.
  • Polyglot: Supports multiple programming languages (Node.js, Python, Java, C#, Go, Ruby, PowerShell).

Use Cases: Backend for web/mobile apps, data processing, real-time file processing, chatbots, IoT backends.

15.6 Amazon API Gateway

Amazon API Gateway is a fully managed service that makes it easy for developers to create, publish, maintain, monitor, and secure APIs at any scale. It acts as a "front door" for applications to access data, business logic, or functionality from your backend services.

Key Features:

  • API Creation: Build RESTful APIs and WebSocket APIs.
  • Traffic Management: Handle traffic management, authorization, and access control.
  • Security: Integrates with IAM, Cognito, and custom authorizers.
  • Scalability: Handles millions of API calls.
  • Integration: Can integrate with AWS Lambda, EC2, HTTP endpoints, and other AWS services.

API Gateway is commonly used to expose Lambda functions as HTTP endpoints, creating serverless APIs.
