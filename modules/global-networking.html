Module 17: Global Networking

17.1 Amazon CloudFront

Amazon CloudFront is a fast content delivery network (CDN) service that securely delivers data, videos, applications, and APIs to customers globally with low latency, high transfer speeds, all within a developer-friendly environment. CloudFront uses a global network of Edge Locations (Points of Presence) to cache content closer to your users.

How CloudFront Works:

  1. A user requests content (e.g., an image, a web page).
  2. The DNS query is routed to the nearest CloudFront Edge Location.
  3. If the content is cached at the Edge Location, it's delivered directly to the user (low latency).
  4. If not cached, CloudFront retrieves the content from your origin server (e.g., S3 bucket, EC2 instance, ELB), caches it, and then delivers it to the user.

Key Benefits:

  • Low Latency: Content delivered from Edge Locations closer to users.
  • High Transfer Speeds: Optimized network paths.
  • Security: Integrates with AWS WAF, Shield, and SSL/TLS.
  • Cost Optimization: Reduces load on origin servers and can lower data transfer costs.
  • Global Reach: Extensive network of Edge Locations worldwide.

17.2 AWS Global Accelerator

AWS Global Accelerator is a networking service that improves the availability and performance of your applications with local or global users. It uses the AWS global network to route user traffic to the optimal application endpoint, regardless of where your users are located.

How Global Accelerator Works:

  1. Global Accelerator provides you with two static Anycast IP addresses that act as a fixed entry point to your application.
  2. User traffic enters the AWS global network at the Edge Location closest to the user.
  3. Global Accelerator then routes the traffic over the highly optimized AWS global network to your application endpoints (e.g., ELBs, EC2 instances) in the nearest healthy AWS Region.

Key Benefits:

  • Improved Performance: Uses the AWS global network for faster routing, bypassing congested public internet paths.
  • Increased Availability: Automatically reroutes traffic to the next healthy endpoint if an endpoint becomes unhealthy.
  • Static IP Addresses: Provides fixed entry points for your application, simplifying DNS management.
  • Traffic Dialing: Allows you to control the percentage of traffic directed to different endpoints, useful for A/B testing or blue/green deployments.

CloudFront vs. Global Accelerator:

  • CloudFront: Best for caching static content (images, videos, web files) at Edge Locations to reduce latency for content delivery.
  • Global Accelerator: Best for improving the performance and availability of dynamic content and applications by routing traffic over the AWS global network to the optimal application endpoint.
