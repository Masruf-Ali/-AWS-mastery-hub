Module 20: Monitoring

20.1 What is Monitoring in AWS?

Monitoring in AWS involves collecting and analyzing data from your applications and infrastructure to understand their health, performance, and resource utilization. Effective monitoring is crucial for identifying issues, optimizing costs, and ensuring the reliability and availability of your services.

Key Aspects of Monitoring:

  • Metrics: Numerical data points that represent the performance or health of a resource (e.g., CPU utilization, network I/O, request count).
  • Logs: Records of events that occur within your applications and infrastructure.
  • Traces: End-to-end views of requests as they flow through distributed systems.
  • Alarms: Notifications triggered when a metric crosses a predefined threshold.

AWS provides several services to help you monitor your cloud environment.

20.2 Amazon CloudWatch

Amazon CloudWatch is a monitoring and observability service that provides you with data and actionable insights to monitor your applications, respond to system-wide performance changes, and optimize resource utilization. CloudWatch collects monitoring and operational data in the form of logs, metrics, and events, providing you with a unified view of AWS resources, applications, and services that run on AWS and on-premises servers.

Key Features:

  • Metrics: Collects metrics from AWS services (EC2, RDS, S3, etc.) and custom metrics from your applications.
  • Logs: Centralizes logs from various sources (EC2 instances, Lambda functions, CloudTrail).
  • Alarms: Create alarms based on metric thresholds to trigger notifications (SNS) or actions (Auto Scaling, EC2 actions).
  • Dashboards: Create custom dashboards to visualize your metrics and logs.
  • Events: CloudWatch Events (now EventBridge) delivers a near real-time stream of system events that describe changes in AWS resources.

20.3 AWS CloudTrail

AWS CloudTrail is a service that enables governance, compliance, operational auditing, and risk auditing of your AWS account. CloudTrail records API calls and related events made by or on behalf of your AWS account and delivers log files to an Amazon S3 bucket.

Key Features:

  • API Call Logging: Records all API calls made to your AWS account.
  • Event History: Provides a 90-day event history of management events.
  • Trails: Create trails to deliver events to an S3 bucket for long-term storage and analysis.
  • Security Analysis: Helps identify who did what, when, and from where in your AWS account.
  • Compliance: Essential for meeting various compliance requirements.

CloudWatch vs. CloudTrail:

  • CloudWatch: Monitors performance and operational health of resources. Answers "How is my application performing?"
  • CloudTrail: Records API activity and changes to your AWS resources. Answers "Who made this change?" or "What happened to this resource?"

20.4 Amazon EventBridge

Amazon EventBridge (formerly CloudWatch Events) is a serverless event bus that makes it easier to connect applications together using data from your own applications, integrated SaaS applications, and AWS services. It delivers a near real-time stream of system events that describe changes in AWS resources.

Key Features:

  • Event Bus: A central hub for routing events.
  • Event Sources: AWS services, custom applications, SaaS partners.
  • Rules: Define patterns to match events and route them to targets.
  • Targets: AWS Lambda, SNS, SQS, Step Functions, EC2 instances, etc.
  • Schema Registry: Helps manage event schemas.

Use Cases: Building event-driven architectures, reacting to changes in AWS resources, integrating applications.

20.5 AWS X-Ray

AWS X-Ray helps developers analyze and debug distributed applications, such as those built using a microservices architecture. With X-Ray, you can understand how your application and its underlying services are performing to identify and troubleshoot the root cause of performance issues and errors.

Key Features:

  • Service Map: Visualizes the components of your application and their connections.
  • Traces: Provides end-to-end views of requests as they travel through your application.
  • Segments: Detailed information about each component in a trace (e.g., compute, database calls, HTTP requests).
  • Annotations & Metadata: Add custom information to traces for easier filtering and analysis.
  • Integration: Works with EC2, Lambda, ECS, Elastic Beanstalk, and other AWS services.

Use Cases: Performance analysis, debugging, identifying bottlenecks in distributed applications.
