Module 4: Networking Basics

4.1 The OSI Model

The Open Systems Interconnection (OSI) model is a conceptual framework that describes functions of a networking or telecommunication system in terms of seven layers. It helps standardize communication protocols and understand how data travels across networks.

  1. Physical Layer (Layer 1): Deals with the physical connection between devices, including cables, connectors, and electrical signals. (e.g., Ethernet cables, Wi-Fi).
    • Units: Bits
  2. Data Link Layer (Layer 2): Provides node-to-node data transfer and handles error correction from the physical layer. This layer has two sub-layers: Media Access Control (MAC) and Logical Link Control (LLC). (e.g., MAC addresses, Switches).
    • Units: Frames
  3. Network Layer (Layer 3): Responsible for logical addressing (IP addresses) and routing of packets across different networks. (e.g., IP, Routers, ARP).
    • Units: Packets
  4. Transport Layer (Layer 4): Provides reliable or unreliable data transfer between end systems, including segmentation, flow control, and error recovery. (e.g., TCP, UDP, Ports).
    • Units: Segments (TCP), Datagrams (UDP)
  5. Session Layer (Layer 5): Establishes, manages, and terminates connections (sessions) between applications. (e.g., NetBIOS, RPC).
    • Units: Data
  6. Presentation Layer (Layer 6): Translates data between the application layer and the network format, handling data encryption, compression, and format conversion. (e.g., JPEG, MPEG, SSL/TLS).
    • Units: Data
  7. Application Layer (Layer 7): Provides network services directly to end-user applications. (e.g., HTTP, FTP, SMTP, DNS).
    • Units: Data

In AWS, you'll primarily interact with components that operate at Layer 3 (Network) and Layer 4 (Transport) for services like EC2, VPC, Load Balancers, and Security Groups, but understanding all layers helps diagnose issues.

4.2 IP Addressing

An Internet Protocol (IP) address is a numerical label assigned to each device connected to a computer network that uses the Internet Protocol for communication. It serves two main functions: host or network interface identification and location addressing.

IPv4 vs. IPv6

  • IPv4: Uses 32-bit addresses (e.g., 192.168.1.1), providing approximately 4.3 billion unique addresses. Due to scarcity, IPv4 addresses are typically managed with Network Address Translation (NAT).
  • IPv6: Uses 128-bit addresses (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334), providing a virtually unlimited number of unique addresses.

Public vs. Private IP Addresses

  • Public IP: Routable over the internet. Devices with public IPs can communicate directly with other devices on the internet.
  • Private IP: Used within a private network (like your home network or an AWS VPC). Not routable over the internet and can be reused in different private networks. Common private IP ranges include:
    • 10.0.0.0 - 10.255.255.255 (Class A)
    • 172.16.0.0 - 172.31.255.255 (Class B)
    • 192.168.0.0 - 192.168.255.255 (Class C)

CIDR (Classless Inter-Domain Routing)

CIDR is a method for allocating IP addresses and for IP routing. It adds flexibility to address allocation compared to the old classful addressing system. A CIDR block is represented by an IP address followed by a slash and a number (e.g., 192.168.1.0/24).

  • The number after the slash (e.g., /24) represents the number of bits in the network portion of the address (i.e., the network mask).
  • A /24 means the first 24 bits are for the network, and the remaining 8 bits are for host addresses (2^8 = 256 addresses).
  • In AWS VPCs, you define your VPC and subnet IP ranges using CIDR blocks.

4.3 Subnets

A subnet (subnetwork) is a logical subdivision of an IP network. Subnetting divides a larger network into smaller, more manageable subnetworks. This helps improve network performance and security.

Why Use Subnets?

  • Organization: Group resources logically (e.g., web servers in one subnet, database servers in another).
  • Security: Apply different security policies (e.g., Network ACLs, Security Groups) to different subnets.
  • Performance: Reduce broadcast traffic within each segment.
  • Availability Zones: In AWS, subnets are always deployed within a single Availability Zone to ensure high availability across zones.

Public vs. Private Subnets (in AWS VPC)

  • Public Subnet: Contains instances that need to be directly accessible from the internet. They have a route to an Internet Gateway.
  • Private Subnet: Contains instances that are isolated from the internet. They can access the internet via a NAT Gateway (for outbound connections), but cannot be directly accessed from the internet. Databases and application servers are typically placed in private subnets.

4.4 DNS (Domain Name System)

DNS is a hierarchical and decentralized naming system for computers, services, or any resource connected to the Internet or a private network. It translates human-readable domain names (e.g., www.example.com) into machine-readable IP addresses (e.g., 192.0.2.1).

How DNS Works (Simplified):

  1. User types a domain name into a web browser.
  2. The local DNS resolver (often provided by your ISP) receives the request.
  3. If the resolver doesn't have the IP, it queries a Root DNS server.
  4. The Root DNS server refers to the Top-Level Domain (TLD) DNS server (e.g., .com).
  5. The TLD DNS server refers to the authoritative DNS server for the domain (e.g., example.com).
  6. The authoritative DNS server provides the IP address for www.example.com.
  7. The IP address is returned to the user's browser, which then connects to the web server at that IP.

AWS offers Route 53, a highly available and scalable cloud DNS web service.

4.5 Firewalls and Security Groups

Firewalls are network security systems that monitor and control incoming and outgoing network traffic based on predetermined security rules. They act as a barrier between trusted and untrusted networks.

In AWS:

  • Security Groups: Act as virtual firewalls for your EC2 instances. They operate at the instance level and control inbound and outbound traffic based on rules (protocol, port range, source/destination IP). Security groups are stateful (meaning if you allow outbound traffic, the return inbound traffic is automatically allowed).
  • Network ACLs (Access Control Lists): Act as optional stateless firewalls for your subnets. They control traffic at the subnet level and have separate inbound and outbound rules that must be explicitly defined for both directions.

4.6 VPN (Virtual Private Network)

A VPN extends a private network across a public network and enables users to send and receive data across shared or public networks as if their computing devices were directly connected to the private network.

Uses of VPN in AWS:

  • Site-to-Site VPN: Connects your on-premises network (data center or office) to your Amazon VPC. This allows resources in your VPC and on-premises network to communicate as if they were in the same private network.
  • Client VPN: Allows individual users to securely connect from their device to their AWS resources and on-premises networks.
  • AWS Client VPN: A managed client-based VPN service that enables you to securely access your AWS resources and resources in your on-premises network from any location.
