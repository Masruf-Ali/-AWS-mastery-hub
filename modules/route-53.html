Module 16: Amazon Route 53

16.1 What is Route 53?

Amazon Route 53 is a highly available and scalable cloud Domain Name System (DNS) web service. It is designed to give developers and businesses an extremely reliable and cost-effective way to route end users to Internet applications by translating human readable names, like www.example.com, into the numeric IP addresses, like 192.0.2.1, that computers use to connect to each other.

Key Features:

  • Domain Registration: Register new domain names directly with Route 53.
  • DNS Service: Manage DNS records for your domains.
  • Traffic Management: Route traffic to various resources using different routing policies.
  • Health Checks: Monitor the health of your resources and route traffic away from unhealthy endpoints.
  • Integration: Integrates seamlessly with other AWS services.

16.2 Route 53 Record Types

Route 53 supports various DNS record types, each serving a specific purpose.

  • A Record (Address Record): Maps a domain name to an IPv4 address (e.g., example.com to 192.0.2.1).
  • AAAA Record (IPv6 Address Record): Maps a domain name to an IPv6 address.
  • CNAME Record (Canonical Name Record): Maps an alias domain name to another canonical domain name (e.g., www.example.com to example.com). Cannot be used for the root domain.
  • Alias Record: An AWS-specific extension to DNS. Maps a domain name to an AWS resource (e.g., ELB, CloudFront distribution, S3 bucket for static website hosting). Can be used for the root domain and behaves like an A record but points to an AWS resource.
  • MX Record (Mail Exchange Record): Specifies the mail servers responsible for receiving email messages on behalf of a domain name.
  • PTR Record (Pointer Record): Used for reverse DNS lookups (mapping an IP address to a domain name).
  • NS Record (Name Server Record): Delegates a DNS zone to use a specific authoritative name server.
  • SOA Record (Start of Authority Record): Provides authoritative information about a DNS zone, such as the primary name server and the email of the domain administrator.
  • TXT Record (Text Record): Stores arbitrary text strings, often used for SPF (Sender Policy Framework) or domain verification.

Alias Records vs. CNAME Records: Alias records are preferred for AWS resources because they can be used for the root domain (e.g., example.com) and are free of charge when pointing to AWS resources.

16.3 Route 53 Routing Policies

Route 53 offers various routing policies to control how DNS queries are responded to, enabling advanced traffic management scenarios.

  • Simple Routing: Routes traffic to a single resource. No health checks.
  • Failover Routing: Routes traffic to a primary resource when it's healthy, and to a secondary resource when the primary is unhealthy. Requires health checks.
  • Geolocation Routing: Routes traffic based on the geographic location of your users.
  • Geoproximity Routing: Routes traffic based on the geographic location of your users and your resources. You can optionally route more traffic to resources in one location than to resources in another.
  • Latency-based Routing: Routes traffic to the resource that provides the best latency for the user.
  • Weighted Routing: Routes traffic to multiple resources based on a weight you assign to each resource. Useful for A/B testing or distributing traffic across different versions of an application.
  • Multivalue Answer Routing: Returns up to eight healthy records selected at random. Useful when you want to return multiple values, such as IP addresses for web servers, and you want to use Route 53 health checking to ensure that only healthy records are returned.

16.4 TTL (Time To Live)

TTL (Time To Live) is a setting in a DNS record that tells a DNS resolver how long to cache a query before requesting a new update. A shorter TTL means changes to your DNS records propagate faster, but it also means more frequent queries to Route 53.

  • Short TTL (e.g., 60 seconds): Good for resources that might change frequently or during planned maintenance/failovers. Faster propagation of changes.
  • Long TTL (e.g., 3600 seconds): Good for stable resources that rarely change. Reduces the load on DNS servers and can improve performance by reducing DNS lookup times.

Alias records in Route 53 ignore the TTL set on the record and behave as if they have a TTL of 0, meaning they are always resolved directly to the AWS resource.

16.5 Route 53 Failover

Route 53 failover routing allows you to configure active-passive or active-active failover configurations for your applications. This enhances the availability and resilience of your services.

How Failover Works:

  1. You create health checks that monitor the health of your primary resources (e.g., an EC2 instance, an ELB, or an endpoint).
  2. You configure a failover routing policy with a primary and a secondary record.
  3. If the health check for the primary resource fails, Route 53 automatically routes traffic to the secondary resource.
  4. When the primary resource becomes healthy again, Route 53 can be configured to automatically fail back to the primary.

This is a critical component for building highly available and fault-tolerant architectures on AWS.
