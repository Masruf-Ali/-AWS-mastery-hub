Module 5: Amazon Virtual Private Cloud (VPC)

5.1 What is a VPC?

Amazon Virtual Private Cloud (VPC) lets you provision a logically isolated section of the AWS Cloud where you can launch AWS resources in a virtual network that you define. You have complete control over your virtual networking environment, including selection of your own IP address range, creation of subnets, and configuration of route tables and network gateways.

Key Concepts:

  • Logically Isolated: Your VPC is completely separate from other virtual networks in the AWS Cloud.
  • Customizable: You can choose your own IP address range (using CIDR notation), create subnets, and control traffic flow.
  • Secure: VPC provides a secure environment for your resources, with features like security groups and network access control lists (NACLs).

5.2 Core VPC Components

A VPC is made up of several key components that work together to create a functional virtual network.

Components:

  • Subnets: A range of IP addresses in your VPC. Subnets are used to partition the network inside a VPC. They can be public (with access to the internet) or private (without direct access to the internet).
  • Route Tables: A set of rules, called routes, that are used to determine where network traffic is directed. Each subnet in your VPC must be associated with a route table.
  • Internet Gateway (IGW): A horizontally scaled, redundant, and highly available VPC component that allows communication between instances in your VPC and the internet.
  • NAT Gateway (Network Address Translation): Enables instances in a private subnet to connect to the internet or other AWS services, but prevents the internet from initiating a connection with those instances.
  • Security Groups: Act as a virtual firewall for your instances to control inbound and outbound traffic. They are stateful.
  • Network Access Control Lists (NACLs): An optional layer of security for your VPC that acts as a firewall for controlling traffic in and out of one or more subnets. They are stateless.

5.3 How to Set Up a VPC

Setting up a VPC involves several steps, from creating the VPC itself to configuring its components.

Basic Setup Steps:

  1. Create the VPC: In the AWS Management Console, navigate to the VPC service and start the VPC wizard or create a VPC manually. You'll need to specify a name and an IPv4 CIDR block for your VPC.
  2. Create Subnets: Once the VPC is created, you can create one or more subnets within it. Each subnet must have its own CIDR block, which is a subset of the VPC's CIDR block.
  3. Create an Internet Gateway: If you need to connect your VPC to the internet, create an Internet Gateway and attach it to your VPC.
  4. Configure Route Tables: Create a custom route table or modify the main route table. To provide internet access to a subnet, add a route to the Internet Gateway (0.0.0.0/0 for IPv4).
  5. Launch Instances: You can now launch EC2 instances and other AWS resources into your subnets.

5.4 Uses of a VPC

VPCs are a fundamental building block for a wide range of applications and services on AWS.

Common Use Cases:

  • Hosting a Public-Facing Website: You can create a VPC with a public subnet to host a web server that is accessible from the internet.
  • Hosting a Multi-Tier Web Application: For a typical three-tier application, you can have a public subnet for your web servers and private subnets for your application servers and databases.
  • Extending Your Corporate Network to the Cloud: You can connect your on-premises data center to your VPC using a VPN connection or AWS Direct Connect, creating a hybrid cloud environment.
  • Disaster Recovery: You can use a VPC as a disaster recovery site for your on-premises infrastructure.
