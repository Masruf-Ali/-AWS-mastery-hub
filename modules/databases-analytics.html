Module 13: Databases & Analytics

13.1 Amazon Relational Database Service (RDS)

Amazon RDS makes it easy to set up, operate, and scale a relational database in the cloud. It provides cost-efficient and resizable capacity while automating time-consuming administration tasks such as hardware provisioning, database setup, patching, and backups. It supports several popular database engines.

Supported Database Engines:

  • Amazon Aurora (AWS proprietary)
  • PostgreSQL
  • MySQL
  • MariaDB
  • Oracle Database
  • Microsoft SQL Server

Key Features:

  • Managed Service: AWS handles patching, backups, and replication.
  • Scalability: Easily scale compute and storage resources up or down.
  • High Availability: Multi-AZ deployments for automatic failover.
  • Read Replicas: Improve read performance and offload read traffic from the primary instance.
  • Automated Backups: Point-in-time recovery.

13.2 Amazon Aurora

Amazon Aurora is a MySQL and PostgreSQL-compatible relational database built for the cloud, combining the performance and availability of traditional enterprise databases with the simplicity and cost-effectiveness of open-source databases. It is a fully managed service that is part of Amazon RDS.

Key Features:

  • High Performance: Up to 5x faster than standard MySQL and 3x faster than standard PostgreSQL.
  • High Availability: Designed for 99.99% availability, automatically replicates data across 3 Availability Zones.
  • Scalability: Storage scales automatically up to 128 TB. Compute can be scaled up or down.
  • Cost-Effective: Pay-as-you-go, often more cost-effective than commercial databases.
  • Serverless Option: Aurora Serverless automatically starts up, shuts down, and scales capacity up or down based on your application's needs.

13.3 Amazon DynamoDB

Amazon DynamoDB is a fast and flexible NoSQL database service for all applications that need consistent, single-digit-millisecond latency at any scale. It is a fully managed, multi-Region, multi-master, durable database with built-in security, backup and restore, and in-memory caching for internet-scale applications.

Key Features:

  • NoSQL: Supports document and key-value data models.
  • Serverless: No servers to provision, patch, or manage.
  • Scalability: Scales automatically to handle any amount of traffic.
  • High Performance: Consistent single-digit-millisecond latency.
  • Global Tables: Multi-Region, multi-master replication.
  • On-demand or Provisioned Capacity: Choose your billing model based on predictability of workload.

Use Cases: Web, mobile, gaming, ad tech, IoT, and many other applications that need low-latency data access.

13.4 Amazon Redshift

Amazon Redshift is a fully managed, petabyte-scale data warehouse service in the cloud. It is optimized for data warehousing workloads, enabling you to run complex analytical queries against large datasets.

Key Features:

  • Columnar Storage: Stores data in a columnar format, which is highly optimized for analytical queries.
  • Massively Parallel Processing (MPP): Distributes and parallelizes queries across multiple nodes for fast performance.
  • Scalability: Easily scale your data warehouse capacity.
  • Cost-Effective: Significantly lower cost than traditional data warehouses.
  • Integration: Integrates with other AWS services like S3, Kinesis, and Glue.

Use Cases: Business intelligence, operational analytics, big data analytics.

13.5 Amazon Athena

Amazon Athena is an interactive query service that makes it easy to analyze data directly in Amazon S3 using standard SQL. Athena is serverless, so there is no infrastructure to set up or manage, and you pay only for the queries you run.

Key Features:

  • Serverless: No servers to manage.
  • Pay-per-query: You pay for the amount of data scanned per query.
  • Standard SQL: Use familiar SQL to query data.
  • Data in S3: Queries data directly from S3, supporting various formats (CSV, JSON, ORC, Parquet).
  • Use Cases: Ad-hoc querying, log analysis, reporting.

13.6 AWS Glue

AWS Glue is a fully managed extract, transform, and load (ETL) service that makes it easy for customers to prepare and load their data for analytics. It discovers your data, transforms it, and makes it available for search and analysis.

Key Features:

  • Serverless ETL: No infrastructure to manage.
  • Data Catalog: A central metadata repository for all your data assets.
  • Crawlers: Automatically discover data and infer schemas.
  • ETL Jobs: Generate, modify, and run ETL scripts (Python or Scala).
  • Integration: Works with S3, RDS, Redshift, and other data stores.

Use Cases: Data integration, data preparation for analytics, building data lakes.

13.7 Amazon EMR (Elastic MapReduce)

Amazon EMR is a managed cluster platform that simplifies running big data frameworks, such as Apache Hadoop and Apache Spark, on AWS to process vast amounts of data. It makes it easy to set up, operate, and scale your big data environments.

Key Features:

  • Managed Hadoop/Spark: Simplifies deployment and management of big data frameworks.
  • Scalability: Easily scale clusters up or down.
  • Cost-Effective: Integrates with EC2 Spot Instances for cost savings.
  • Integration: Works with S3, DynamoDB, and other AWS data stores.

Use Cases: Big data processing, machine learning, ETL, scientific simulation.
