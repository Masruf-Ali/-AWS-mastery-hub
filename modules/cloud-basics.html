Module 2: Cloud Basics

2.1 Evolution of Computing

To understand cloud computing, it's helpful to look at how computing infrastructure has evolved over time.

On-Premises (Traditional IT)

Historically, organizations hosted their applications and data in their own data centers. This involved significant upfront investment in hardware, software, networking, and facilities. Companies were responsible for everything from physical security to application maintenance.

  • Pros: Full control, high customization.
  • Cons: High cost, slow provisioning, maintenance burden, scalability challenges, disaster recovery complexity.

Virtualization

Virtualization emerged as a key technology to improve efficiency in on-premises environments. It allows a single physical server to run multiple isolated virtual machines (VMs), each with its own operating system and applications. This led to better hardware utilization and easier resource management.

  • Pros: Better resource utilization, easier server provisioning, reduced hardware costs.
  • Cons: Still requires significant upfront investment and management of physical infrastructure.

Cloud Computing

Cloud computing takes virtualization a step further by abstracting away the underlying physical infrastructure entirely. Resources are provided as a service over the internet, managed by a third-party provider (like AWS). Users only pay for what they consume.

  • Pros: Scalability, elasticity, cost-effectiveness, agility, global reach, reduced operational burden.
  • Cons: Requires internet connectivity, potential vendor lock-in, security concerns (though often exaggerated).

2.2 Virtualization Deep Dive

Virtualization is the creation of a virtual (rather than actual) version of something, such as a server, desktop, storage device, network, or even an operating system. It's the fundamental technology that underpins cloud computing.

How Virtualization Works: Hypervisors

At the core of virtualization is the hypervisor (also known as a Virtual Machine Monitor or VMM). A hypervisor is a piece of software, firmware, or hardware that creates and runs virtual machines (VMs).

  • Type 1 Hypervisor (Bare-Metal): Runs directly on the host hardware to control the hardware and to manage guest operating systems. Examples include VMware ESXi, Microsoft Hyper-V, and Citrix XenServer. These are commonly used in data centers and cloud environments.
  • Type 2 Hypervisor (Hosted): Runs on a conventional operating system (OS) just like other computer programs. Guest OSs run as a process on the host OS. Examples include VMware Workstation, Oracle VirtualBox. These are typically used for end-user virtualization on personal computers.

AWS primarily uses a modified Xen hypervisor and more recently, the KVM-based Nitro hypervisor for its EC2 instances.

2.3 IT vs. Cloud: A Comparison

Let's compare the traditional IT model with cloud computing across several dimensions:

Feature Traditional IT (On-Premises) Cloud Computing
Cost Model Capital Expenditure (CapEx) - large upfront investment Operational Expenditure (OpEx) - pay-as-you-go
Scalability Limited, requires manual provisioning, slow Highly elastic, scales on demand, fast
Maintenance Full responsibility for hardware, software, facilities Managed by cloud provider (shared responsibility)
Agility/Speed Slow provisioning of resources (weeks/months) Rapid provisioning of resources (minutes)
Global Reach Requires building data centers in each region Global infrastructure available instantly
Disaster Recovery Complex and expensive to implement Easier and more cost-effective to implement
Security Full control and responsibility Shared responsibility model

2.4 Cloud Service Models: IaaS, PaaS, SaaS

Cloud computing services are broadly categorized into three main service models, each offering different levels of management and flexibility.

Infrastructure as a Service (IaaS)

IaaS provides the fundamental building blocks of cloud IT. It gives you the highest level of flexibility and management control over your IT resources. You manage the operating system, applications, and data, while the cloud provider manages the underlying infrastructure (virtualization, servers, storage, networking).

  • Examples: Amazon EC2, Amazon S3, Amazon VPC.
  • You Manage: Applications, Data, Runtime, OS.
  • Provider Manages: Virtualization, Servers, Storage, Networking.

Platform as a Service (PaaS)

PaaS removes the need for you to manage the underlying infrastructure (hardware and operating systems) and allows you to focus on the deployment and management of your applications. The cloud provider manages the operating system, database, and web servers.

  • Examples: AWS Elastic Beanstalk, AWS Lambda (partially), Heroku.
  • You Manage: Applications, Data.
  • Provider Manages: Runtime, OS, Virtualization, Servers, Storage, Networking.

Software as a Service (SaaS)

SaaS provides you with a complete product that is run and managed by the service provider. You simply use the software as an end-user. You don't worry about how the service is maintained or how the underlying infrastructure is managed.

  • Examples: Gmail, Salesforce, Dropbox, AWS S3 (as a storage service for end-users).
  • You Manage: Nothing (just use the application).
  • Provider Manages: Applications, Data, Runtime, OS, Virtualization, Servers, Storage, Networking.

2.5 Cloud Deployment Models

Cloud computing offers various deployment models to suit different organizational needs and strategies.

Public Cloud

In a public cloud model, computing services are delivered over the public internet and are available to anyone who wants to purchase them. Resources are shared among multiple tenants, but logically isolated.

  • Characteristics: Shared infrastructure, pay-as-you-go, high scalability, managed by provider.
  • Examples: AWS, Microsoft Azure, Google Cloud Platform.

Private Cloud

A private cloud refers to cloud computing resources used exclusively by a single business or organization. A private cloud can be physically located on the company's on-site data center, or it can be hosted by a third-party service provider.

  • Characteristics: Dedicated resources, higher control, enhanced security, higher cost.
  • Examples: VMware Cloud on AWS, AWS Outposts (extends AWS infrastructure to on-premises).

Hybrid Cloud

A hybrid cloud combines a public cloud with a private cloud, allowing data and applications to be shared between them. This model offers greater flexibility and more deployment options.

  • Characteristics: Mix of public and private, flexibility, data portability, cost optimization.
  • Examples: Using AWS for burst capacity while keeping sensitive data on-premises.
