Module 18: DevOps & Infrastructure as Code (IaC)

18.1 What is DevOps?

DevOps is a set of practices that combines software development (Dev) and IT operations (Ops) to shorten the systems development life cycle and provide continuous delivery with high software quality. It aims to unify software development and software operation.

Key Principles of DevOps:

  • Culture: Collaboration, communication, and shared responsibility.
  • Automation: Automate repetitive tasks (build, test, deploy).
  • Continuous Integration (CI): Developers frequently merge code changes into a central repository, where automated builds and tests are run.
  • Continuous Delivery (CD): Code changes are automatically built, tested, and prepared for a release to production.
  • Continuous Deployment: Code changes are automatically deployed to production after passing all tests.
  • Monitoring and Logging: Continuous monitoring of application performance and infrastructure health.

18.2 Infrastructure as Code (IaC)

Infrastructure as Code (IaC) is the management of infrastructure (networks, virtual machines, load balancers, etc.) in a descriptive model, using the same versioning as DevOps team uses for source code. Like the principle that the same source code generates the same binary, an IaC model generates the same environment every time it is applied.

Benefits of IaC:

  • Consistency: Ensures environments are identical across development, testing, and production.
  • Automation: Eliminates manual configuration and human error.
  • Version Control: Infrastructure configurations are versioned and auditable.
  • Speed: Rapid provisioning of infrastructure.
  • Cost Reduction: Reduces operational overhead.

AWS offers several services to implement IaC and DevOps practices.

18.3 AWS CloudFormation

AWS CloudFormation is a service that helps you model and set up your Amazon Web Services resources so that you can spend less time managing those resources and more time focusing on your applications that run in AWS. You create a template that describes all the AWS resources that you want (e.g., EC2 instances, RDS databases, S3 buckets), and CloudFormation provisions and configures those resources for you.

Key Features:

  • Declarative Language: Define your infrastructure in JSON or YAML templates.
  • Automated Provisioning: CloudFormation handles the order of provisioning and dependencies.
  • Version Control: Templates can be stored in version control systems.
  • Drift Detection: Detects if your stack's resources have been manually modified outside of CloudFormation.
  • Rollback Capabilities: Automatically rolls back changes if an error occurs during deployment.

Use Cases: Deploying entire application stacks, creating consistent environments, disaster recovery setup.

18.4 AWS Elastic Beanstalk

AWS Elastic Beanstalk is an easy-to-use service for deploying and scaling web applications and services developed with Java, .NET, PHP, Node.js, Python, Ruby, Go, and Docker on familiar servers such as Apache, Nginx, Passenger, and IIS. You simply upload your code, and Elastic Beanstalk automatically handles the deployment, from capacity provisioning, load balancing, auto-scaling to application health monitoring.

Key Features:

  • Platform as a Service (PaaS): Abstract away infrastructure management.
  • Supports Multiple Languages/Platforms: Wide range of supported environments.
  • Automated Deployment: Handles provisioning and configuration of underlying resources (EC2, S3, ELB, RDS).
  • Health Monitoring: Provides dashboards and metrics for application health.
  • Customizable: You can still access and customize the underlying resources if needed.

Use Cases: Rapid deployment of web applications, development and test environments, small to medium-sized applications.

18.5 AWS CodePipeline

AWS CodePipeline is a continuous delivery service that helps you automate your release pipelines for fast and reliable application and infrastructure updates. It automates the build, test, and deploy phases of your release process every time there is a code change, based on the release model you define.

Key Stages:

  • Source: Retrieves source code from repositories (e.g., CodeCommit, GitHub, S3).
  • Build: Compiles code, runs unit tests (e.g., CodeBuild).
  • Test: Runs integration or system tests (e.g., CodeBuild, CodeDeploy).
  • Deploy: Deploys the application to target environments (e.g., CodeDeploy, Elastic Beanstalk, CloudFormation).

18.6 AWS CodeDeploy

AWS CodeDeploy is a service that automates code deployments to any instance, including Amazon EC2 instances, on-premises servers, serverless Lambda functions, and Amazon ECS services. It makes it easier to rapidly release new features, helps avoid downtime during application deployment, and handles the complexity of updating your applications.

Key Features:

  • Automated Deployments: Automates application deployments.
  • In-place and Blue/Green Deployments: Supports various deployment strategies to minimize downtime.
  • Rollbacks: Automatically rolls back deployments on failure.
  • Integration: Works with CodePipeline, GitHub, Bitbucket, and S3.

Use Cases: Deploying applications to EC2, Lambda, or ECS, automating software releases.
