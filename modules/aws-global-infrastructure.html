Module 6: AWS Global Infrastructure

6.1 Regions

An AWS Region is a physical location in the world where AWS has multiple Availability Zones. Each Region is a separate geographic area, designed to be completely isolated from other Regions. This isolation achieves the greatest possible fault tolerance and stability.

  • Global Presence: AWS has dozens of Regions around the world (e.g., us-east-1 for N. Virginia, eu-west-1 for Ireland, ap-southeast-2 for Sydney).
  • Data Residency: You choose a Region to host your services, and your data will reside within that Region. This is crucial for data governance and regulatory compliance.
  • Latency: Deploying resources closer to your users reduces network latency.
  • Cost: Pricing for AWS services can vary by Region.

When you launch an instance, you select an AWS Region. All resources within that Region are then provisioned there.

6.2 Availability Zones (AZs)

An Availability Zone (AZ) is one or more discrete data centers with redundant power, networking, and connectivity in an AWS Region. AZs are physically separated by a meaningful distance (typically many kilometers) from other AZs, though all are within 100 km (60 miles) of each other. This physical separation helps ensure that events like fires, floods, or power outages in one AZ do not affect other AZs.

  • High Availability: By distributing your applications across multiple AZs within a Region, you can achieve high availability and fault tolerance. If one AZ goes down, your application can continue to run in another AZ.
  • Low Latency: AZs in a Region are connected to each other with high-bandwidth, low-latency networking over fully redundant, dedicated metro fiber.
  • Minimum per Region: Each AWS Region has at least two Availability Zones, and many have three or more.

6.3 Edge Locations and Regional Edge Caches

Edge Locations are sites that Amazon CloudFront (AWS's Content Delivery Network) uses to cache copies of your content closer to your customers. This reduces latency for end-users accessing your content.

  • Global Distribution: There are significantly more Edge Locations than Regions.
  • Content Delivery: Primarily used for services like CloudFront, Route 53, and AWS Shield.
  • Regional Edge Caches: Located between your origin servers and the global Edge Locations. They have larger cache sizes than individual Edge Locations and serve as a second tier of caching, further improving performance for less frequently accessed content.

6.4 Shared Responsibility Model (Recap)

As discussed in Module 3, the Shared Responsibility Model is fundamental to understanding security in AWS. AWS is responsible for the security OF the cloud, while the customer is responsible for security IN the cloud.

  • AWS Responsibilities: Global infrastructure (Regions, AZs, Edge Locations), physical security of data centers, hardware, software, networking that runs AWS services.
  • Customer Responsibilities: Customer data, operating systems, applications, network and firewall configurations, platform management, identity and access management.

This model ensures that both parties contribute to the overall security posture of cloud deployments.

6.5 AWS Pricing Fundamentals

AWS pricing is based on a pay-as-you-go model, meaning you only pay for the services you consume. There are no upfront costs or long-term contracts required (unless you opt for them for discounts).

Key Principles:

  • Pay for what you use: You are charged for the compute, storage, and other resources you consume.
  • Pay less when you reserve: For predictable workloads, reserving capacity (e.g., Reserved Instances for EC2) can significantly reduce costs.
  • Pay less with volume discounts: As your usage increases, the per-unit cost often decreases.
  • Pay less as AWS grows: AWS continuously reduces prices as it achieves greater economies of scale.
  • Custom pricing: For very large enterprises, custom pricing agreements may be available.

Factors Influencing Cost:

  • Compute: Instance type, operating system, software, Region, purchase model (On-Demand, Reserved, Spot).
  • Storage: Storage class, amount of storage, data transfer out, number of requests.
  • Data Transfer: Data transferred out of AWS Regions is generally charged; data transferred in is usually free. Data transfer between AWS services within the same Region is often free or very low cost.

Always use the AWS Pricing Calculator and set up billing alerts (as discussed in Module 1) to manage and monitor your costs effectively.
