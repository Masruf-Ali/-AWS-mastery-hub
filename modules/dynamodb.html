Module 14: Amazon DynamoDB

14.1 What is DynamoDB?

Amazon DynamoDB is a fully managed, serverless, key-value NoSQL database designed to run high-performance applications at any scale. DynamoDB offers built-in security, continuous backups, automated multi-region replication, in-memory caching, and data export tools.

Key Characteristics:

  • NoSQL Database: DynamoDB is a non-relational database, storing data in key-value pairs or as documents.
  • Fully Managed: AWS handles all the operational overhead, including hardware provisioning, setup, configuration, replication, software patching, and cluster scaling.
  • Scalable: Provides seamless scalability with consistent, single-digit millisecond latency.
  • Flexible Data Model: Supports both key-value and document data models.

14.2 Core DynamoDB Concepts

DynamoDB has a few core components that you need to understand to use it effectively.

  • Tables: A collection of data.
  • Items: A collection of attributes. Each item is uniquely identifiable by a primary key. This is similar to a row or a record in other database systems.
  • Attributes: A fundamental data element, something that does not need to be broken down any further. This is similar to a column or a field in other database systems.
  • Primary Key: Uniquely identifies each item in a table. The primary key can be a simple primary key (partition key only) or a composite primary key (partition key and sort key).

14.3 How to Set Up a DynamoDB Table

Setting up a DynamoDB table is straightforward using the AWS Management Console, CLI, or SDKs.

Basic Setup Steps:

  1. Navigate to DynamoDB: In the AWS Management Console, go to the DynamoDB service.
  2. Create Table: Click the "Create table" button.
  3. Table Name: Provide a name for your table.
  4. Primary Key: Define the primary key for your table. You will specify the attribute name and type for the partition key, and optionally for the sort key.
  5. Capacity Mode: Choose a capacity mode: On-demand or Provisioned.
  6. Create: Click "Create" to create the table.

14.4 DynamoDB Read/Write Capacity Modes

DynamoDB offers two capacity modes to manage the cost and performance of your tables.

  • On-Demand: DynamoDB instantly accommodates your workloads as they ramp up or down. You pay for what you use, making it a good option for unpredictable workloads.
  • Provisioned: You specify the number of reads and writes per second that you require for your application. You can use Auto Scaling to automatically adjust your tableâ€™s capacity based on traffic. This is a good option for predictable workloads.

14.5 Uses of DynamoDB

DynamoDB is a good fit for a variety of applications that require low-latency data access at any scale.

Common Use Cases:

  • Web and Mobile Applications: User profiles, session data, and shopping carts.
  • Gaming: Leaderboards, player data, and game state.
  • IoT: Sensor data, device metadata, and real-time data ingestion.
  • Real-time Bidding: Ad targeting, campaign data, and user profiles.
