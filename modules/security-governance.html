Module 24: Security & Governance

24.1 AWS WAF (Web Application Firewall)

AWS WAF is a web application firewall that helps protect your web applications or APIs from common web exploits that may affect availability, compromise security, or consume excessive resources. WAF gives you control over how traffic reaches your applications by enabling you to create security rules that block common attack patterns, such as SQL injection or cross-site scripting, and filter traffic based on conditions that you specify, such as IP addresses, HTTP headers, and HTTP body or URI strings.

Key Features:

  • Protects Web Applications: Filters malicious traffic before it reaches your application.
  • Customizable Rules: Create your own rules based on IP addresses, HTTP headers, HTTP body, URI strings, SQL injection, XSS.
  • Managed Rules: Use AWS Managed Rules for common threats.
  • Integration: Integrates with CloudFront, Application Load Balancer (ALB), and API Gateway.

24.2 AWS Shield

AWS Shield is a managed Distributed Denial of Service (DDoS) protection service that safeguards applications running on AWS. AWS Shield provides always-on detection and automatic inline mitigations that minimize application downtime and latency.

Types of AWS Shield:

  • AWS Shield Standard: Automatically included with all AWS accounts at no additional cost. Provides protection against common, most frequent DDoS attacks.
  • AWS Shield Advanced: A paid service that provides enhanced protections for applications running on EC2, Elastic Load Balancing (ELB), Amazon CloudFront, AWS Global Accelerator, and Route 53. Includes DDoS cost protection, 24/7 access to the AWS DDoS Response Team (DRT), and advanced metrics and reports.

Use Cases: Protecting web applications and APIs from DDoS attacks.

24.3 AWS Key Management Service (KMS)

AWS Key Management Service (KMS) makes it easy for you to create and manage cryptographic keys and control their use across a wide range of AWS services and in your applications. KMS is a secure and resilient service that uses hardware security modules (HSMs) to protect your keys.

Key Features:

  • Centralized Key Management: Create, store, and manage encryption keys.
  • Integration with AWS Services: Seamlessly integrates with S3, EBS, RDS, Lambda, and many other services for encryption.
  • Auditability: All API calls to KMS are logged in CloudTrail.
  • Customer Master Keys (CMKs): You can create and manage your own CMKs, or use AWS-managed CMKs.

Use Cases: Encrypting data at rest and in transit, managing secrets, digital signatures.

24.4 Amazon GuardDuty

Amazon GuardDuty is a threat detection service that continuously monitors your AWS accounts and workloads for malicious activity and unauthorized behavior. GuardDuty uses machine learning, anomaly detection, and integrated threat intelligence to identify and prioritize potential threats.

Key Features:

  • Continuous Monitoring: Monitors AWS CloudTrail event logs, VPC Flow Logs, and DNS logs.
  • Threat Intelligence: Uses continuously updated threat intelligence feeds.
  • Machine Learning: Detects anomalies and suspicious behavior.
  • Findings: Generates detailed security findings that can be integrated with other security services.

Use Cases: Detecting compromised EC2 instances, unauthorized API activity, port scanning, crypto-mining.

24.5 Amazon Macie

Amazon Macie is a data security and data privacy service that uses machine learning and pattern matching to discover and protect your sensitive data in AWS. Macie automatically discovers, classifies, and reports on sensitive data stored in Amazon S3.

Key Features:

  • Sensitive Data Discovery: Identifies personally identifiable information (PII) and other sensitive data in S3 buckets.
  • Data Classification: Classifies data based on its content and sensitivity.
  • Security Alerts: Generates alerts for potential data security risks.
  • Visibility: Provides a dashboard to visualize your S3 data security posture.

Use Cases: Data privacy compliance (GDPR, HIPAA), sensitive data protection, security auditing.

24.6 Amazon Inspector

Amazon Inspector is an automated security assessment service that helps improve the security and compliance of applications deployed on AWS. Inspector automatically assesses applications for vulnerabilities and deviations from best practices.

Key Features:

  • Automated Security Assessments: Scans EC2 instances and container images for vulnerabilities.
  • Vulnerability Management: Identifies software vulnerabilities and unintended network exposure.
  • Compliance Checks: Assesses compliance with security best practices and standards.
  • Detailed Reports: Provides prioritized lists of security findings with remediation steps.

Use Cases: Vulnerability scanning, compliance auditing, security posture management.

24.7 AWS Secrets Manager

AWS Secrets Manager helps you protect access to your applications, services, and IT resources by enabling you to easily rotate, manage, and retrieve database credentials, API keys, and other secrets throughout their lifecycle.

Key Features:

  • Automated Rotation: Automatically rotates secrets (e.g., database credentials) without interrupting applications.
  • Centralized Management: Store and manage all your secrets in one place.
  • Fine-grained Access Control: Control access to secrets using IAM policies.
  • Auditability: All access to secrets is logged in CloudTrail.
  • Integration: Integrates with RDS, Redshift, DocumentDB, and other services.

Use Cases: Managing database credentials, API keys, OAuth tokens, and other sensitive information.
