Interview Prep: Module 6: AWS Global Infrastructure

Here are some common interview questions related to the AWS Global Infrastructure module, along with suggested answers.

1. What is an AWS Region, and why is it important to choose the right one?

Answer: An AWS Region is a physical location in the world where AWS clusters data centers. Each Region is isolated for fault tolerance. Choosing the right Region is critical for data residency (compliance), latency (proximity to users), and cost (pricing varies by Region). It also impacts the availability of certain services.

2. Explain the concept of an Availability Zone (AZ) and how it contributes to high availability.

Answer: An Availability Zone (AZ) is one or more discrete data centers within an AWS Region, with redundant power, networking, and connectivity. AZs are physically separated to prevent a single event (like a fire or flood) from affecting multiple AZs. Deploying applications across multiple AZs within a Region ensures high availability and fault tolerance, as if one AZ fails, the application can continue to run in another.

3. What are AWS Edge Locations and how do they differ from Regions and AZs?

Answer: Edge Locations are sites used by Amazon CloudFront (CDN) to cache content closer to end-users, reducing latency. They are distinct from Regions and AZs, which host core AWS services and infrastructure. Edge Locations are more numerous and geographically dispersed than Regions, focusing on content delivery rather than full service deployment.

4. Briefly explain the AWS Shared Responsibility Model in the context of global infrastructure.

Answer: AWS is responsible for the security OF the Cloud, which includes the global infrastructure (Regions, AZs, Edge Locations), physical security of data centers, and the underlying hardware and software. The customer is responsible for security IN the Cloud, which includes their data, applications, operating systems, and network configurations deployed within that global infrastructure.

5. How does AWS pricing work, and what are some ways to optimize costs?

Answer: AWS pricing is primarily pay-as-you-go, meaning you only pay for the services you consume. Costs are influenced by compute, storage, and data transfer. To optimize: use the AWS Free Tier, choose appropriate instance types and storage classes, leverage Reserved Instances or Savings Plans for predictable workloads, utilize Spot Instances for fault-tolerant tasks, and monitor usage with AWS Cost Explorer and billing alerts.

6. What is the significance of data residency when choosing an AWS Region?

Answer: Data residency refers to the physical location where data is stored. It's significant because many countries and industries have strict regulations (e.g., GDPR, HIPAA) that require data to be stored within specific geographic boundaries. Choosing the correct AWS Region ensures compliance with these legal and regulatory requirements.

7. How does AWS ensure low latency for users accessing content globally?

Answer: AWS ensures low latency through its extensive global infrastructure. This includes: deploying applications in AWS Regions geographically close to users, utilizing Edge Locations for caching static content via CloudFront, and leveraging AWS Global Accelerator to route dynamic traffic over the optimized AWS global network, bypassing congested public internet paths.

8. Can you explain the difference between an Edge Location and a Regional Edge Cache?

Answer: Both are part of CloudFront's network. Edge Locations are the most numerous and geographically dispersed points where content is cached and delivered to users. Regional Edge Caches are larger caches located between Edge Locations and your origin servers. They hold less frequently accessed content than Edge Locations but more than origin servers, serving as a second tier of caching to further reduce latency and origin load.

9. What are some factors that can influence the cost of AWS services?

Answer: Factors include: Service type (e.g., EC2, S3, RDS), resource size/capacity (e.g., instance type, storage amount), Region (pricing varies), data transfer out (egress costs), data requests (e.g., S3 PUT/GET requests), purchase model (On-Demand, Reserved, Spot), and additional features like data encryption or monitoring.

10. Why does AWS recommend using IAM users instead of the root account for daily tasks?

Answer: The root account has unrestricted access to all resources and services in the AWS account. Using it for daily tasks increases the risk of accidental misconfigurations or security breaches. IAM users allow for the principle of least privilege, where users are granted only the permissions necessary to perform their specific tasks, significantly enhancing security and auditability.
