Interview Prep: Module 8: EC2

Here are some common interview questions related to the EC2 module, along with suggested answers.

1. What is Amazon EC2, and what are its core benefits?

Answer: Amazon EC2 (Elastic Compute Cloud) provides resizable compute capacity in the cloud, essentially virtual servers (instances). Core benefits include: Elasticity (scale up/down easily), Cost-effectiveness (various pricing models), Control (full control over instances), Integration (seamless with other AWS services), and Reliability (runs on Amazon's proven infrastructure).

2. Describe the different EC2 instance types and when you would use each.

Answer: EC2 offers various instance types optimized for different workloads:

  • General Purpose (M, T): Balanced CPU, memory, networking. Good for web servers, dev/test (e.g., T2/T3 for burstable, M5 for general).
  • Compute Optimized (C): High-performance processors. For HPC, batch processing, gaming servers.
  • Memory Optimized (R, X, Z): High memory-to-CPU ratio. For high-performance databases, in-memory caches.
  • Storage Optimized (I, D, H): High sequential read/write access to local storage. For NoSQL DBs, data warehousing.
  • Accelerated Computing (P, G, F): Hardware accelerators (GPUs). For ML, graphics-intensive apps.

3. Explain the different EC2 pricing models and when to use each.

Answer:

  • On-Demand: Pay by hour/second, no commitment. For irregular, unpredictable workloads, dev/test.
  • Reserved Instances (RIs): Commit for 1 or 3 years. Significant discount for steady-state, predictable workloads.
  • Savings Plans: Flexible commitment (1 or 3 years) for consistent usage ($/hour). Discounts across EC2, Fargate, Lambda.
  • Spot Instances: Bid on unused capacity, up to 90% off. For fault-tolerant, flexible, stateless workloads (can be interrupted).
  • Dedicated Hosts: Physical servers for your use. For licensing requirements, strict compliance.

4. What is an EC2 Security Group, and how does it function?

Answer: An EC2 Security Group acts as a virtual firewall for your EC2 instances, controlling inbound and outbound traffic. It operates at the instance level, is stateful (return traffic is automatically allowed), and only supports ALLOW rules. All inbound traffic is denied by default, and all outbound traffic is allowed by default. You define rules based on protocol, port range, and source/destination IP addresses or other security groups.

5. How do you securely connect to a Linux EC2 instance? What is an EC2 Key Pair?

Answer: You securely connect to a Linux EC2 instance using SSH (Secure Shell) and an EC2 Key Pair. An EC2 Key Pair consists of a public key (stored on AWS and associated with the instance) and a private key (stored securely on your local machine). When you connect, the private key authenticates you to the instance. It's crucial to keep the private key file (`.pem`) secure and never share it.

6. What is an Amazon Machine Image (AMI), and why would you create a custom one?

Answer: An AMI provides the information needed to launch an instance, including a template for the root volume (OS, application server, applications), launch permissions, and block device mappings. You would create a custom AMI to: pre-configure instances with your specific software and settings, speed up deployment, ensure consistency across instances, and create a golden image for your applications.

7. Describe a scenario where Spot Instances would be a good choice.

Answer: Spot Instances are excellent for fault-tolerant, flexible, and stateless workloads that can tolerate interruptions. Examples include: batch processing jobs (e.g., image rendering, video encoding), data analysis, containerized applications (where tasks can be easily restarted), and development/testing environments where cost savings are prioritized over strict uptime guarantees.

8. How would you ensure that your EC2 instances are highly available?

Answer: To ensure high availability, deploy EC2 instances across multiple Availability Zones (AZs) within a Region. Use an Elastic Load Balancer (ELB) to distribute traffic across these instances. Implement an Auto Scaling Group (ASG) to automatically replace unhealthy instances and scale capacity up or down based on demand, ensuring a desired number of healthy instances are always running.

9. What is User Data when launching an EC2 instance? Provide an example.

Answer: User Data is a script or set of commands that you can provide when launching an EC2 instance. This script runs automatically the first time the instance starts up. It's used for bootstrapping tasks like installing software, updating packages, configuring services, or downloading files. For example, a user data script could install Apache web server and start it: `#!/bin/bash sudo yum update -y sudo yum install -y httpd sudo systemctl start httpd sudo systemctl enable httpd`

10. What is the difference between a Public IP and an Elastic IP for an EC2 instance?

Answer:

  • Public IP: Automatically assigned to an EC2 instance upon launch if in a public subnet. It changes every time the instance is stopped and started.
  • Elastic IP (EIP): A static, public IPv4 address that you can allocate to your AWS account and associate with an EC2 instance. It remains associated with your account until you release it, even if the instance is stopped or terminated. EIPs are used for applications that require a fixed public IP address.
